# ADR 0009: Outcome-first integrated view

## Context

The final application combines many concepts, but displaying every panel simultaneously undermines the learning progression.

## Decision

Chapter 10 shows the terminal outcome and limitation first. Plan, evidence, events, usage and evaluation are collapsed sections backed by one integrated Python application service.

## Alternatives

- A permanent multi-panel operations dashboard.
- Separate mock data for the final UI.
- Hide all technical detail behind one answer.

## Consequences

Learners can inspect depth without facing it all at once, and the final demonstration cannot drift from the tested engine.
